19. Особенности машинной арифметики

Системы счисления

Обычное представление числа: x=±an...a1a0.b1...bm...x=\pm a_n...a_1a_0.b_1...b_m...x=±an​...a1​a0​.b1​...bm​...﻿ По сути его можно представить в виде: an∗10n+...+a1∗101+a0∗100+b1∗10−1+....a_n*10^{n}+...+a_1*10^1+a_0*10^0+b_1*10^{-1}+....an​∗10n+...+a1​∗101+a0​∗100+b1​∗10−1+....﻿ - позиционная система счисления. ЭВМ использует двоичную систему счисления. поэтому такое предоставление в ней выглядит так: an∗2n+...+a1∗21+a0∗20+b1∗2−1+....a_n*2^{n}+...+a_1*2^1+a_0*2^0+b_1*2^{-1}+....an​∗2n+...+a1​∗21+a0​∗20+b1​∗2−1+....﻿

Для хранения в машине числа используют поле фиксированной длины - машинное слово. Выделяют по форме представления, мпособу хранения и реализации арифметических операций целые и вещественные числа

Представления целых чисел

Вообщем, для представляется целое число таким способом:

x=aL∗2L+aL−1∗2L−1+....+a0∗20x=a_L*2^L+a_{L-1}*2^{L-1}+....+a_0*2^0x=aL​∗2L+aL−1​∗2L−1+....+a0​∗20﻿. L в данном случае какое-то фиксированное число. Все для хранения числа отводят L+2 разрядов, так как нужен 1 разряд еще на знак. Получается, что существкет какое-то пороговое число xmax=2L+1−1x_{max}=2^{L+1}-1xmax​=2L+1−1﻿. Если при выполнении арифметических операций получилось число по модулю большее этого, то могут возникнуть проблемы.

Представление вещественных чисел.

Число с плавающей точкой представляется в виде: x=±(y1∗2−1+...+yt∗2−t)∗2p,y1,...,yt−двоичные цифрыx=\pm(y_1*2^{-1}+...+y_t*2^{-t})*2^p, y_1,...,y_t - двоичные \ цифрыx=±(y1​∗2−1+...+yt​∗2−t)∗2p,y1​,...,yt​−двоичные цифры﻿. μ=(y1∗2−1+...+yt∗2−t)\mu = (y_1*2^{-1}+...+y_t*2^{-t})μ=(y1​∗2−1+...+yt​∗2−t)﻿ - мантисса числа, t-разрядность мантиссы. p - двоичный порядок. p записывают также как двоичное число, для хранения которого в машинном коде отводится l+2 двоичных разрядов.
